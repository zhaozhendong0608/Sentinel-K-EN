# Landscape

> This file is the core entry point for the AI to understand the project. Please keep it updated.

## 1. Project Metadata
- **Project Name**: Sentinel-K (Sentinel Kernel)
- **Core Goal**: Build a "collaborative operating system" specifically designed for highly efficient AI-human collaboration, achieving engineering-level controlled execution, auditing, and asset management.
- **Tech Stack**: 
  - **Core Protocol**: Workflow instruction set based on GitHub Flavored Markdown.
  - **Layered Structure**: Kernel Layer + Workflows Layer + Body Layer + Archive Layer.
  - **Collaboration Tool**: Deeply integrates the interaction loop between AI Agent (Execution) and Human (Commander).
- **Architecture Pattern**: Intent-Driven -> Atomic Execution -> Evidence Verification.

## 2. Core Business Landscape
- **Kernel Layer**:
  - **SSOT**: `00_Kernel` defines the supreme constitution.
  - **Asset Grading**: Classifies and governs `Solid` (strictly regulated) and `Liquid` (fluid) assets.
  - **System Boot**: Physically isolates hallucinations through layered data packet loading strategies like `Pack-0/1/2`.
- **Workflow Layer (SOPs)**:
  - **Full Lifecycle SOPs**: Includes `/01_System_Boot`, `/02_Planning_Radar`, `/03_Execution_Sentry`, `/04_Quality_Gate`, `/05_Archiving_Reset`.
  - **Anti-Hallucination Boot**: Built-in self-evaluation model and confidence score card (`[K-SCORE]`).
- **Execution & Evidence Layer (Body & Evidence)**:
  - **Pizza Logic (`[K-PIZZA]`)**: Forced breakdown of complex tasks into Bone/Stub/Muscle stages.
  - **Evidence Chain Driven**: Deliverables must include `Verification Evidence` to guarantee traceability.

## 3. Current Milestone
- **Phase**: Kernel 2.0 Industrialization Alignment
- **Focus**: Improving multi-language specification comparison tables, strengthening the physical execution of Pizza Logic, and improving machine readability of evidence blocks.

## 4. Known Issues & Tech Debt
- **Context Bloat**: As project complexity increases, Pack loading needs to be more precise (introducing Lazy Loading mechanisms).
- **Cross-language Adaptation**: Specification details for languages like Python/Go are still being completed (refer to `02_Specs`).
- **Hallucination Noise**: During long sessions, the AI may drift away from overall constraints, requiring periodic `/05_Archiving_Reset`.
